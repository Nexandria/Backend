# Proyecto

Nexandria

## Stack

- NestJS
- PostgreSQL
- Prisma

## Principios

- Clean Architecture
- SOLID
- TypeScript estricto
- DTOs obligatorios
- Validación con class-validator

## Antes de programar

1. Analizar el código existente.
2. No asumir implementaciones.
3. Reutilizar componentes existentes.
4. Mantener consistencia con la arquitectura actual.

## Backend

- Controllers sin lógica.
- Services contienen reglas de negocio.
- Repositories manejan acceso a datos.

## Frontend

- React Hook Form.
- Zod.
- TanStack Query.
- Componentes reutilizables.

## Al finalizar

- Ejecutar tests relevantes.
- Actualizar documentación si corresponde.
